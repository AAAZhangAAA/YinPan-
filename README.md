> 本说明长期有效，每次更新资源后照着做即可。

## 二、添加资源的三种方式（任选）

### 方式 A：后台表单 + 让助手部署（最省事，推荐）
1. 打开 `/admin.html`，输入密码进入。
2. 填表添加（标题/简介/分类/网盘链接/提取码…），可点「+ 再添加一个网盘链接」加多个。
3. 点「添加到列表」暂存（可连续加多条）。
4. 全部加完后点「生成并复制」，把得到的 `data.js` 文本**粘到和助手的对话里**，说一句「更新并重新部署」。
5. 助手写进文件并重发 CloudStudio，粉丝立刻能看到。

### 方式 B：后台表单 + 自己部署
1. 同上 1–3 步填表。
2. 点「下载 data.js」得到文件。
3. 用下载的 `data.js` **覆盖**项目里的 `assets/js/data.js`。
4. 按第四节重新部署。

### 方式 C：直接手改 data.js（熟练后最干净）
- 用任意文本编辑器打开 `assets/js/data.js`。
- 在 `RESOURCES = [ ... ]` 数组里，照第五节模板加一条（复制一条改内容即可）。
- 保存文件 → 按第四节部署。

> ⚠️ 后台的「本机暂存」只存在当前浏览器（localStorage）。**添加与导出最好在同一次、同一个页面里完成**，否则暂存可能丢失。

## 三、重新部署到网页

### 让助手部署（最简单）
在对话里说「重新部署」，助手用 CloudStudio 工具一键发布，无需你操作任何文件。

### 自己部署（需一次性接好托管）
- 当前站点托管在 **CloudStudio**。若想完全自己管理，可把整个项目文件夹接到
  **GitHub Pages / Netlify / Vercel** 等静态托管（一次配置），之后每次改完 `data.js`
  重新上传/推送，托管会自动更新，比每次找助手更顺手。
- **本地预览**（仅你自己本机看）：在项目目录运行
  ```bash
  python -m http.server 8000
  ```
  浏览器打开 http://localhost:8000 即可。别人打不开这个地址。

## 五、data.js 单条资源模板

复制下面这段，改掉内容即可（`id` 别和已有的重复）：

```js
{
  id: "d-001",                              // 唯一编号，建议 d=纪录片/m=电影/t=电视剧/v=综艺/p=插件
  title: "资源标题",
  brief: "一句话简介（卡片上显示）",
  description: "详情页完整描述，可写多段。",
  category: "纪录片",                      // 必须和分类同名：纪录片/电影/电视剧/综艺/插件工具
  subcategory: "自然",                      // 可选，二级分类（如 自然/历史/科学/美食 等）
  cover: "",                               // 留空=自动生成占位图；或填图片URL / assets/img/xxx.jpg
  tags: ["自然", "4K"],                     // 标签，用于搜索
  recommended: true,                       // true = 进首页「推荐」区
  date: "2026-07-12",                      // 更新日期，最新更新区按它排序
  links: [
    { name: "夸克网盘", url: "https://你的真实网盘链接", password: "提取码" },
    { name: "百度网盘", url: "https://...", password: "" }   // password 留空字符串 = 无需提取码
  ]
},
```

- **封面图**：留空自动生成带分类的渐变占位图；想用真图就填图片网址，或把图片放进 `assets/img/` 写相对路径。
- **网盘链接/密码**：换成你真实的夸克/百度/阿里等链接；密码留空 `""` 表示无提取码。

## 六、修改后台密码

1. 打开 `/admin.html` 登录页，底部点「修改访问密码」。
2. 输入新密码 → 点「生成哈希」→ 复制生成的那串字符。
3. 打开 `assets/js/admin.js`，把顶部 `ADMIN_HASH = "..."` 里的旧哈希替换成新的。
4. 重新部署生效。

---

## 七、重要说明

- 网站**完全公开**，无登录限制。粉丝只能看、不能写；写入需要后台密码 + 部署权限（仅你持有），所以「只能你添加」在发布环节被强制保证。
- 真实的影视/插件文件仍在百度、夸克等网盘上，本站只做**索引与展示**，不占用服务器空间和流量。
- 后台密码门是前端防护（deterrent），不是军事级安全；若以后需要更强控制，可加真后端鉴权。
- 修改分类体系：编辑 `assets/js/data.js` 顶部的 `CATEGORIES` 数组（名称 + 二级分类 `subs`），并在 `app.js` 的 `COVER_PALETTE` 里补上对应配色即可。
